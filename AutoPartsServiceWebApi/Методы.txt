Обзор API

1. Authenticate
1.1 POST Login (на вход подается номер телефона, на выходе получаем информацию новый ли пользователь true-false и смс код). 
1.2 POST Authenticate (на вход подается номер телефона и смс код, на выходе получаем токен + если пользователь новый - то сообщение "This is a new user")
1.3 POST Register (на вход подаются регистрационные данные пользователя, jwt, на выходе создается новый пользователь). В поле UserType нужно ввести Common или Business для определения типа пользователя при регистрации). 
1.4 GET userInfo (на вход jwt и deviceid(на данные момент генерируется при регистрации, в будущем поменяем), на выходе информация о пользователе)

2. User
2.1 POST addCar (по id UserCommon мы добавляем информацию о машине и загружаем в бд)
2.2 PUT UserCommon/{id} (изменение данных пользователя, подаем на вход измененные данные, на выходе обновленный пользователь)
2.3 DELETE /Car/{carId} (на входе userId и carId, на выходе удаленное авто)
2.4 POST {userId}/Service (по userId мы добавляем сервис для сущности UserBusiness)
2.5 DELETE /Service/{id} (на вход Id service, на выходе удаленная сущность сервиса)
2.6 GET /Service/{id} (на вход id, на выходе информация о сервисе)
2.7 GET /GetDocuments/{userId} (по userid получаем информацию о документах)
2.8 POST AddDocument/{userId} (по userId добавляем документ)
2.9 POST CheckFines (надо доделать реализацию самой проверки)
2.10 POST AddReview/{userId} (по userid добавляем отзыв + оценку пользователю)

3. Request
3.1 POST Request/{userCommonId} (указываем userCommonId, на вход подаем информацию о заявке, она помечается в базе как Active true)
3.2 GET Request/active (получаем из БД список всех активных заявок)
3.3 POST accept/{requestId}/{userId} (указываем requestId - запрос который мы хотим подтвердить, userId - пользователь, который подтверждает заявку)
